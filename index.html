02 —

Problems to Solve

🔍
Discoverability
Collections are siloed across institutions. Users can't explore connections between Oxford and Berlin holdings — coins, manuscripts, and museum objects exist in separate islands.
🗺️
No Geographic Context
Artefacts from the Ethnological Museum and coin collections have geographic origins but there is no visual map-based interface to see where objects came from.
🕸️
Hidden Provenance Networks
The Culture Translocated dataset contains 3,872 entities linked to acquisitions — but no tool exists to visualise the web of relationships between collectors, dealers, and the museum.
🔤
Inaccessible Typefaces
327 digitised type specimen books are largely unsearchable beyond basic metadata. The actual typeface designs are locked inside image files with limited OCR.
🎵
Under-used Audio Heritage
110 early Berlin recordings sit as raw audio files with no interactive or analytical interface to explore the music, its patterns, or its cultural context.
📜
Manuscript Data Hard to Query
The Bodleian's medieval manuscript TEI catalogue is rich but requires XML expertise to query. Non-technical researchers have no accessible entry point.
03 —

Proposed Systems

01
Beginner–Intermediate
Cross-Collection Explorer Map
An interactive world map showing where cultural objects came from — bridging Oxford and Berlin institutions on a single geographic canvas.
What it builds

An interactive QGIS / Leaflet-powered map that plots artefacts from the Ashmolean coin collection, the Ethnological Museum entities dataset, and the Bate Collection instruments by their geographic origin. Users click a region to browse related objects with images via IIIF.


Datasets Used
Ancient Coin Data Culture Translocated Ashmolean Metadata IIIF Bodleian
Tools
QGIS / Leaflet.js OpenRefine Python Allmaps

Key Features
  • Filter by institution, time period, object type
  • Click pins to preview object images via IIIF API
  • Heat-map layer showing acquisition density by region
  • Export filtered results as CSV

Solves the geographic discoverability gap — letting anyone see the physical world these collections come from, not just abstract catalogue numbers.
02
Intermediate
Provenance Network Visualiser
A graph-based network tool that maps the human and institutional networks behind colonial-era museum acquisitions.
What it builds

Using the Culture Translocated dataset (3,872 entities), build a network graph with Gephi or D3.js showing relationships between collectors, dealers, government agencies, and when/how artefacts entered the Ethnological Museum. Reveals patterns in provenance chains.


Datasets Used
Culture Translocated Ashmolean Metadata
Tools
Gephi igraph (Python) D3.js OpenRefine

Key Features
  • Node types: person, company, society, government
  • Timeline slider to animate network growth by decade
  • Highlight "hub" entities — most connected acquirers
  • Search any entity to trace their full network

Directly supports repatriation research and provenance transparency — one of the most pressing issues in GLAM today.
03
Beginner–Friendly
WW1 Voices: Named Entity Explorer
An NLP-powered tool that surfaces people, places and events buried inside 'Lest We Forget' crowdsourced archive records.
What it builds

Apply Named Entity Recognition (NER) and Keyword Extraction to the Lest We Forget JSON dataset. Build a simple web interface that lets users search by person name, place, regiment or date — surfacing records that standard metadata search would miss. HuggingFace Spaces tools already exist as a starting point.


Datasets Used
Lest We Forget
Tools
spaCy / HuggingFace crowdsourced-data-tools Google Colab SentenceTransformers

Key Features
  • Auto-tag people, places, dates from free text
  • Semantic search: "find records about Somme" returns contextually related entries
  • Simple map of mentioned locations
  • Export matched records for further research

Makes WW1 personal histories searchable by family members and researchers who currently can't find records without exact names.
04
Advanced
Berlin Typeface Classifier & Browser
A computer vision tool to classify and visually browse 327 digitised type specimen books from 19th–20th century Berlin.
What it builds

Use computer vision (Distant Viewer Explorer / HuggingFace models) to classify type specimens by style — serif, sans-serif, blackletter, decorative. Build a visual browsing interface grouped by typeface family. The OCR fulltext can also be mined to extract foundry names, dates and style descriptions.


Datasets Used
Berlin's Typographic Legacy
Tools
Distant Viewer Explorer HuggingFace Python / scikit-learn OpenRefine

Key Features
  • Visual similarity search: "find fonts like this one"
  • Filter by decade, foundry, style category
  • Side-by-side comparison of specimens
  • Timeline view of how Berlin typography evolved

Unlocks a visually rich archive for designers, historians and typographers — currently inaccessible without manually reading through 20 GB of images.
05
Intermediate
Dining Through Time: Menu Card Data Story
An interactive data visualisation of 2,403 European menu cards from 1880–1913, revealing patterns in cuisine, language, and dining culture.
What it builds

Mine the historical menu card CSV metadata and images to build a scrollable data story: what dishes appeared most? Which languages were used? How did menus change before and after key historical events? Combine with Orange Data Mining for text analysis of menu content.


Datasets Used
On the Menu: Historical Menucards
Tools
Orange Data Mining Python / Pandas D3.js / Plotly Google Colab

Key Features
  • Timeline chart: dish frequency over 30 years
  • Language map: where French, German, English menus were used
  • Word cloud of most common ingredients by decade
  • Image gallery viewer with filter by year/country

Transforms a niche archival collection into an engaging public history tool — accessible to food historians, cultural researchers and curious members of the public.
06
Intermediate
Manuscript Scholar's Query Tool
A no-code query interface over the Bodleian's medieval manuscript TEI catalogue — making 800 years of manuscript history searchable by anyone.
What it builds

Parse the Medieval Manuscripts TEI XML into a flat, queryable dataset using the existing tabular data export. Build a faceted search and filter interface: by language, century, script type, material (vellum/paper), origin country. Optionally connect IIIF images for visual preview.


Datasets Used
Medieval Manuscripts IIIF Bodleian
Tools
Python OpenRefine Gale Digital Scholar Lab IIIF API

Key Features
  • Filter by: language, date range, physical format, origin
  • Natural language query: "show me 13th century English psalters"
  • View manuscript images inline via IIIF
  • Download filtered results as CSV or JSON

Removes the XML barrier — any humanities researcher, not just digital scholars, can now query one of the world's greatest manuscript catalogues.
